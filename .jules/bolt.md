## 2025-01-28 - Memoizing Framer Motion Components in Lists
**Learning:** Components utilizing expensive Framer Motion hooks like `useSpring` and `useTransform` (e.g., `IconContainer` in a dock) suffer from severe performance degradation if they re-render on every parent state change. For instance, the `links` array in `Navbar.tsx` containing JSX was breaking referential equality and causing `FloatingDock` and all `IconContainer` components to re-render whenever the `Navbar` state changed.
**Action:** Always wrap components containing multiple Framer Motion hooks in `React.memo` (using named function expressions to satisfy ESLint, e.g., `memo(function Name(...) {...})`). Furthermore, always `useMemo` array props containing objects with JSX elements or functions that are passed down to memoized components to preserve referential equality.
